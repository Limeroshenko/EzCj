# 詞庫規則：

2字詞：①首尾②首尾

3字詞：①首②首③首尾

4字詞（含）以上：①首②首③首④首

字詞分離互不干擾，所以打完詞後要按「;」出詞

**此3字的取碼與散彈倉頡不同：**

這3個字從我初學電腦時就是錯誤編碼，我就不管這個「事實標準」了。打正確的倉頡碼吧！

①【撐】取：手火月竹（詞庫首尾取「手竹」）

②【捏】取：手日土（詞庫取首尾取「手土」）

③【銹】取：金竹木尸（詞庫取首尾取「金尸」）

當然，打單字的時候，你打原本的碼，還是會出字。

# 表格內容：

## EzCJv2_Ext.cin

大字集的散彈倉頡v2，字元收到unicode 延伸F區。

這是以gcin為出發點思考的，要用在其他平台使用，請自行把「第3欄的註解」與gcin的flag通通刪掉。

## EzCJv2_A.cin

普通字集的散彈倉頡v2，字元收到unicode 延伸A區。但補足教育部本土語言成果參考字。

## EzCJv2（手機版）.cin

給安gcin for android用的，內容與EzCJv2_A.cin完全一樣，但輸入詞組時不用加「;」。我猜這個檔案應該也可以餵給IOS的相關輸入法。

因為虛擬鍵盤的選字是用看的，而不那麼依賴肌肉記憶，用「；」出詞就顯得多餘了。

## EzCJv2.lime

這是給萊姆輸入法用的，內容與EzCJv2（手機版）.cin完全一樣。

## mscj3.json

笑死，這不是微軟倉頡，而是這是給PIME用的，內容與EzCJv2_Ext.cin一致，支援到unicode 延伸F區。在windows我喜歡用PIME打倉頡。

把這個檔案丟進

"C:\Program Files (x86)\PIME\python\cinbase\json"

覆蓋，然後選「倉頡」，原本酷倉裡最難用的微軟倉頡就變成散彈倉頡了。
